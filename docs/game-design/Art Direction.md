# Objetivo

Este documento define a direção artística oficial do Journey World.

Seu propósito é garantir consistência visual em toda a plataforma, independentemente da ferramenta utilizada para produzir os assets.

Todos os ilustradores, designers e inteligências artificiais devem seguir estas diretrizes.

---

# Filosofia

A arte do Journey World deve transmitir acolhimento, curiosidade e alegria.

O objetivo não é impressionar pela complexidade visual, mas criar um universo consistente, memorável e emocionalmente confortável para crianças e suas famílias.

Cada elemento deve parecer pertencer ao mesmo mundo.

---

# Pilares Visuais

Toda arte deve transmitir:

- Alegria
- Leveza
- Segurança
- Descoberta
- Imaginação
- Esperança

Nunca transmitir:

- Medo
- Violência
- Agressividade
- Caos
- Excesso de estímulos

---

# Estilo

O Journey World utiliza um estilo de ilustração 2D estilizado.

Características:

- formas simples;
- cantos arredondados;
- proporções infantis;
- poucos detalhes;
- leitura imediata.

A simplicidade deve facilitar animações e produção de novos conteúdos.

---

# Personagens

Todos os personagens seguem a mesma linguagem visual.

Características:

- cabeça levemente maior;
- olhos expressivos;
- nariz simplificado;
- boca simples;
- mãos simplificadas;
- pés simplificados.

Evitar excesso de detalhes anatômicos.

---

# Proporções

As proporções dos personagens devem permanecer constantes.

Nunca alterar:

- tamanho da cabeça;
- distância entre os olhos;
- comprimento dos braços;
- formato do rosto;
- estrutura corporal.

Toda nova pose deve preservar essas proporções.

---

# Expressões

As emoções devem ser transmitidas principalmente por:

- olhos;
- sobrancelhas;
- boca;
- postura corporal.

Evitar exageros caricatos.

A leitura emocional deve ser imediata.

---

# Paleta de Cores

A paleta prioriza tons suaves e saturação moderada.

Objetivos:

- conforto visual;
- identidade consistente;
- boa leitura.

Evitar:

- neon;
- contraste excessivo;
- excesso de preto puro.

Cada Jornada poderá possuir uma paleta complementar.

---

# Iluminação

Iluminação simples.

Sem efeitos realistas.

Sombras suaves.

Poucos níveis de profundidade.

O foco deve permanecer na legibilidade.

---

# Contornos

Contornos consistentes.

Espessura uniforme.

Evitar linhas extremamente finas ou extremamente grossas.

---

# Texturas

Poucas texturas.

Superfícies limpas.

Evitar ruído visual.

---

# Cenários

Os cenários apoiam a narrativa.

Nunca competem visualmente com o personagem.

Características:

- poucos elementos;
- profundidade simples;
- cores suaves;
- foco na navegação.

---

# Objetos

Objetos seguem o mesmo estilo dos personagens.

Características:

- arredondados;
- simplificados;
- facilmente reconhecíveis;
- poucos detalhes.

---

# Interface

A interface faz parte do mundo.

Não deve parecer separada do restante da experiência.

Botões, cartões e painéis utilizam:

- cantos arredondados;
- sombras discretas;
- poucas cores simultâneas.

---

# Ícones

Ícones devem ser:

- simples;
- preenchidos;
- consistentes;
- facilmente reconhecíveis.

Evitar estilos mistos.

---

# Assets

Todo asset deve ser criado pensando em reutilização.

Sempre que possível:

- personagem separado do fundo;
- objetos independentes;
- camadas reutilizáveis.

---

# Transparência

Todos os personagens, objetos, móveis, roupas, acessórios e elementos interativos devem possuir fundo verdadeiramente transparente.

Não utilizar:

- fundo branco;
- fundo quadriculado;
- transparência simulada.

---

# Formatos

## PNG

Utilizar para:

- personagens;
- roupas;
- pets;
- móveis;
- objetos;
- efeitos;
- adesivos;
- troféus.

PNG é o formato padrão do projeto.

---

## SVG

Utilizar apenas quando:

- o asset for vetorial;
- possuir poucas formas;
- precisar ser escalável.

Exemplos:

- logotipos;
- ícones;
- símbolos;
- formas simples.

Nunca utilizar SVG para personagens completos.

---

# Consistência

Um asset novo deve parecer ter sido criado no mesmo dia que os primeiros assets do projeto.

Mudanças de:

- traço;
- proporção;
- iluminação;
- saturação;
- espessura do contorno;

não são aceitáveis.

---

# Character Bible

Todo personagem principal deve possuir um Character Bible.

Esse documento define:

- proporções;
- cores;
- roupas;
- expressões;
- poses;
- detalhes permanentes.

Nenhuma nova ilustração deve ser produzida sem consultar o Character Bible.

---

# Asset Pipeline

Antes de criar um asset:

1. Consultar Character Bible.
2. Consultar Art Bible.
3. Verificar paleta.
4. Verificar proporções.
5. Gerar asset.
6. Comparar com assets existentes.
7. Aprovar ou revisar.

---

# Uso de Inteligência Artificial

Toda IA deve:

- reutilizar referências oficiais;
- editar personagens existentes sempre que possível;
- evitar recriar personagens do zero;
- preservar identidade visual.

Sempre utilizar imagens de referência oficiais durante a geração.

---

# Princípios

A direção artística do Journey World deve priorizar:

- consistência acima de variedade;
- identidade acima de realismo;
- simplicidade acima de complexidade;
- legibilidade acima de detalhes;
- emoção acima de espetáculo.

O jogador deve reconhecer imediatamente qualquer imagem como pertencente ao universo do Journey World.
