# Journey World
# UI - Design System

**Versão:** 1.0
**Status:** Draft

---

# Objetivo

Este documento define os padrões visuais do Journey World.

Seu propósito é garantir consistência em toda a interface, permitindo que novas telas e componentes reutilizem uma mesma linguagem visual.

O Design System deve transmitir calma, acolhimento, fantasia e crescimento.

---

# Filosofia

Journey World não é um aplicativo corporativo.

Também não é um jogo competitivo.

Ele representa um mundo vivo onde a criança cultiva sua própria história.

Toda decisão visual deve reforçar esta sensação.

---

# Personalidade Visual

A interface deve transmitir:

- Calma
- Acolhimento
- Segurança
- Curiosidade
- Fantasia
- Natureza
- Descoberta
- Esperança

Evitar aparência:

- Técnica
- Corporativa
- Competitiva
- Agressiva
- Exageradamente infantil

---

# Direção Artística

Inspirado em:

- Livros ilustrados
- Contos infantis
- Natureza
- Jardins
- Bosques
- Madeira
- Papel
- Pintura digital
- Aquarela suave

Evitar:

- Neon
- Cyberpunk
- Sci-fi
- Interface futurista
- Visual excessivamente "game"

---

# Tipografia

Priorizar:

- Excelente legibilidade
- Letras arredondadas
- Aparência amigável

Evitar:

- Fontes técnicas
- Fontes condensadas
- Fontes decorativas em excesso

Máximo:

Duas famílias tipográficas.

---

# Espaçamento

Utilizar escala consistente.

Exemplo:

4

8

12

16

24

32

48

64

Nunca utilizar valores aleatórios.

---

# Bordas

Componentes:

12 a 20 px

Cards:

Mais arredondados.

Diálogos:

Arredondamento médio.

Botões:

Sempre arredondados.

---

# Sombras

Sombras suaves.

Pouco contraste.

Sensação de profundidade leve.

Nunca utilizar sombras pesadas.

---

# Ícones

Estilo:

Outline arredondado.

Traço consistente.

Espessura uniforme.

Misturar ícones apenas quando fizer sentido.

---

# Ilustrações

Devem seguir um único estilo artístico.

Sempre utilizar:

- mesmas proporções;
- mesma iluminação;
- mesma direção de pintura;
- mesma paleta.

Nunca misturar estilos.

---

# Personagens

Todos devem compartilhar:

- mesmas proporções;
- mesmo estilo de olhos;
- mesma iluminação;
- mesma espessura de contorno;
- mesma linguagem artística.

---

# Botões

Tipos:

Primary

Secondary

Ghost

Danger

Link

Todos derivados do mesmo componente.

---

# Cards

Todos os cards devem compartilhar:

- borda;
- sombra;
- padding;
- raio;
- comportamento.

O conteúdo muda.

A estrutura permanece.

---

# Estados

Todo componente deve possuir:

Normal

Hover (Web)

Pressed

Focused

Disabled

Loading

Selected

Error

Success

---

# Feedback

Sempre priorizar:

Mudanças sutis.

Pequenas animações.

Pouco movimento.

Evitar excesso de estímulos.

---

# Navegação

A navegação deve ser simples.

Poucos níveis.

Poucas decisões por tela.

A criança nunca deve se sentir perdida.

---

# Responsividade

A interface deve funcionar em:

Celulares

Tablets

Modo retrato

Modo paisagem (futuro)

---

# Acessibilidade

Priorizar:

Bom contraste.

Texto legível.

Botões grandes.

Áreas de toque confortáveis.

Ícones acompanhados por texto quando necessário.

---

# Emojis

Podem ser utilizados apenas quando agregarem significado.

Nunca substituir completamente ícones.

---

# Microinterações

Preferir:

Escala

Fade

Pequeno movimento

Evitar:

Animações longas

Movimentos bruscos

Efeitos chamativos

---

# Princípios

A interface deve desaparecer.

A criança deve prestar atenção na sua jornada, e não no aplicativo.

O Journey World deve transmitir a sensação de um livro de histórias que ganha vida pouco a pouco, preservando memórias e incentivando experiências no mundo real.
